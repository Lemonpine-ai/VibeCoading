## 플로우 화살표(MCP) 한계 — 해결 방법

Talk to Figma MCP의 `create_connections`는 **이미 파일 안에 “기본으로 쓸 커넥터(Connector)” 노드가 있어야** 동작합니다. 처음에는 없어서 오류가 납니다.

### 방법 A — FigJam 커넥터 복사 (MCP로 자동 연결까지 쓰고 싶을 때)

1. **FigJam** 새 파일을 엽니다.
2. 도구에서 **Connector**(화살표)를 하나 그립니다. 스타일·두께는 원하는 대로.
3. 그 커넥터를 **복사(Ctrl/Cmd+C)** 합니다.
4. **다보냥이 있는 Figma 디자인 파일** 같은 페이지에 **붙여넣기(Ctrl/Cmd+V)** 합니다.
5. Cursor에서 Talk to Figma MCP로 **`set_default_connector`** 를 실행할 때, **방금 붙여 넣은 커넥터의 노드 ID**를 넘깁니다.  
   - 플러그인/선택으로 ID 확인 후 MCP에 전달.
6. 그 다음 **`create_connections`** 에 `startNodeId` / `endNodeId` 를 넣으면, 같은 스타일로 프레임 간 연결이 생성됩니다.

### 방법 B — Figma 디자인 파일에서만 수동 (가장 단순)

1. Figma **디자인** 모드에서 **리소스(Shift+I)** 또는 상단 **FigJam** 섹션에서 **Connector**를 씁니다. (없으면 FigJam 파일에서 복사해 오는 A와 동일)
2. 프레임 `3:22` → `3:28` → `4:48` → `4:49` 순으로 **드래그해 연결**합니다.
3. 라벨은 커넥터 선택 후 텍스트 추가.

### 방법 C — 프로토타입 모드

1. **Prototype** 탭에서 각 버튼·탭에 **인터랙션**을 연결해 화면 전환을 만듭니다.  
2. “화살표”는 **플로우 다이어그램**이 아니라 **클릭 플로우**로 검증할 때 유리합니다.

**정리:** MCP로 화살표를 자동으로 쓰려면 **한 번은 커넥터 노드를 페이지에 넣고 `set_default_connector`로 등록**해야 하고, 그게 번거로우면 **방법 B로 수동 연결**하면 됩니다.

권장 연결(참고):
1. `3:22` → `3:28` (시작하기)
2. `3:28` → `4:48` (Health 탭)
3. `4:48` → `4:49` (Account 탭)
